# Project Governance

This document outlines the governance model for the Agent Dashboard project.

## Governance Model: BDFL

The Agent Dashboard project currently follows the **Benevolent Dictator for Life (BDFL)** model.

### Roles

#### Benevolent Dictator for Life (BDFL)
The BDFL (currently **Dustin Black**) has the final say on all project decisions, including:
- Technical direction and architecture.
- Release scheduling and versioning.
- Conflict resolution within the community.
- Appointment and removal of Maintainers.

#### Maintainers
Maintainers are individuals who have demonstrated a long-term commitment to the project and have been granted write access to the repository. Responsibilities include:
- Reviewing and merging pull requests.
- Triage and management of GitHub issues.
- Participating in technical discussions and providing guidance to contributors.

### Decision Making

Most decisions are reached through consensus among Maintainers in public discussions (issues and pull requests). In cases where consensus cannot be reached, the BDFL provides the final decision.

### Evolution of Governance

As the Agent Dashboard community grows and diversifies, this governance model may evolve. Potential future transitions include moving to a **Technical Steering Committee (TSC)** model to better represent a broader set of stakeholders and organizations. Any such change will be discussed openly with the community before being implemented.
